Nillion decentralizes trust for sensitive data in the same way that blockchains decentralized transactions.

We're in the midst of a data renaissance. Emerging ecosystems like personalized AI, decentralized trading and identity risk the safety, power and control of important data by entrusting it to one entity.

The Nillion thesis: Decentralized, private data will be the backbone of tomorrow's biggest industries - opening up a vast white space of web3 use cases and verticals.

  • [AI]

    Nillion enables the training and inference of AI models in a secure and confidential manner, creating the backbone of secure personalized AI.

  • [DePIN]

    Nillion's secure computing network forms a confidential backend for other DePIN networks while creating the foundation for diverse physical resource networks to be built such as in IoT, Agriculture, or Supply chains.

  • [PETs]

    Privacy-Enhancing Technologies built directly into Nillion's infrastructure make it possible to store sensitive, confidential, or otherwise valuable data by distributing trust and relying on the highest cryptographic security standard - Information-Theoretic Security.

[ How it
works ]

[01]

Processing Layer

Nodes are deployed with Nillion's NDK (Node Deployment Kit) and connect to form compute clusters that distribute the trust of securely processing data.

[02]

Coordination Layer

A blockchain execution environment, streamlined to only support intra-node use, facilitates payments, requests, and network resource coordination.

[03]

Connectivity Layer

The network interfaces with external systems and blockchains to expand access to Nillion's compute and storage.

Anatomy of a
Nillion Node

  • [ ] Cryptographic Preprocessor

    Nodes generate the key network resource needed to process data - a type of correlated randomness referred to as a blinding factor - by performing a process known as "preprocessing".

  • [ ] Secure Resource Vault

    Each node stores its share of the blinding factor securely, distributing trust across the network in a quantum-safe way.

  • [ ] Coordination Interface

    The network manages information flow and payments through the Coordination Layer, with nodes reading and writing as needed.

  • [ ] Public Particle Database

    Users store and process their data in the network by masking it with preprocessed blinding factors, generating Information-Theoretic Secure (ITS) particles. These particles are portable so storage solutions are flexible, so long as nodes can access them.

  • [ ] Program Library

    Nodes have access to an ever-expanding library of user-generated programs that can be run over particalized data.

  • [ ] Authentication and Access Control Service

    Nodes handle network authentication and collaboratively manage data access for computation and storage.
